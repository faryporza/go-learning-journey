### 🧠 สรุปแก่นความรู้ (Key Takeaways) ที่ได้จากเรื่องนี้

1. **Race Condition คือฝันร้าย:** การเขียนโค้ดแค่เช็คว่า `if stock > 0` ไม่เพียงพอเมื่อมีคนจำนวนมากยิง Request เข้ามาใน *เสี้ยววินาทีเดียวกัน* เพราะ Database จะล็อคและประมวลผลไม่ทัน
2. **Redis คือ Gatekeeper ที่ยอดเยี่ยม:** เราใช้ Redis มาเป็น "นายประตู" รับแรงกระแทกแทน MySQL คนที่มาคนที่ 11 เป็นต้นไป จะถูก Redis เตะกลับไปทันทีโดยที่ Request นั้นยังไม่ทันไปกวนใจ MySQL เลยด้วยซ้ำ
3. **`DECR` (Atomic Operation):** คำสั่งลดค่าตัวเลขของ Redis ถูกออกแบบมาให้รับประกันว่า "ลดทีละ 1 แน่นอน" ต่อให้คน 10,000 คนสั่งพร้อมกัน ตัวเลขก็จะไม่เพี้ยนและไม่ข้ามคิวกันครับ
